# Jour 01 — Pourquoi le DevOps ?

>  📅 Date : 31 mars 2026 | ✅ Statut : Complété

## 🎯 Concepts Clés

### 1. Le problème avant DevOps
- Équipes Dev et Ops travaillaient en silos → conflits constants
- Déploiements lents et instables
- Mentalité : "Ça marche sur ma machine !"

### 2. Qu'est-ce que le DevOps ?
- DevOps = Development + Operations
- C'est une **culture** et une **philosophie**, pas un outil
- Né en 2009 à Gand, Belgique avec Patrick Debois

### 3. Les principes CALMS
| Principe | Description |
|---------|-------------|
| Culture | Collaboration Dev/Ops, responsabilité partagée |
| Automation | Automatiser tests, déploiements, infra |
| Lean | Éliminer les gaspillages |
| Measurement | Mesurer performances et erreurs |
| Sharing | Partager les connaissances |

### 4. Le cycle DevOps
```
Plan → Code → Build → Test → Release → Deploy → Operate → Monitor → Plan...
```

### 5. Pourquoi c'est essentiel
- Netflix/Amazon déploient des dizaines de fois par jour
- Réduction du Time-To-Market
- Profils DevOps très demandés en 2026

## 💡 Points Retenus
- DevOps = culture avant tout
- L'automatisation est centrale
- Sans DevOps : semaines → Avec DevOps : minutes
